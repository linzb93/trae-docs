# 前端测试用例自动生成专家

## 1. 任务背景

当提供原型图描述或进入新开发分支时，根据当前 Git 分支名及源码逻辑，生成标准化的测试验收清单。

## 2. 目录与命名逻辑

- 获取 Git 分支名（例如 `dev-1.1.0`）。
- 目录名：`__test__/{version}/`（移除 `dev-` 前缀，如 `1.1.0`，如果在`master`或者`main`分支，目录名就为`1.0.0`。）。
- 文件名：生成 `auto.md` (AI 自动检查) 和 `manual.md` (人工自测)。

## 3. 生成准则

### 【自动化用例 - auto.md】

- **核心逻辑**：要求 AI 扫描当前代码变更，对比原型图要求。
- **强制检查项**：
    - **分页组件**：检查是否处理了总条数(Total)、每页数量(PageSize)和跳转逻辑。
    - **表单验证**：检查是否包含输入限制（如：2-20位中英文、11位数字、非空提示）。
    - **空状态**：检查是否实现了 `Empty` 占位或 Loading 态。
- **格式**：
    - [ ] **{组件/功能名}实现检查**：{代码段描述}。如有，请打钩 [x]，否则 [ ]。

### 【手动用例 - manual.md】

- **标题(E)**：验证 [模块] 的 [功能点] 是否符合预期。
- **前置条件(F)**：如“已登录系统”、“未绑定XX账号”。
- **测试步骤(G)**：
    1. 主操作路径。
       1.1 异常操作分支（如不输入内容点击提交）。
- **预期结果(H)**：
    - 界面应显示：[具体文案/弹窗]。
    - 逻辑应完成：[页面跳转/数据刷新]。
- **自测状态**：[ ] 未完成 / [x] 已通过。

## 4. 触发示例

"检测到当前分支是 dev-1.1.0，请根据我刚上传的《订单管理》原型图，在 **test**/1.1.0/ 目录下创建 auto.md 和 manual.md。"
