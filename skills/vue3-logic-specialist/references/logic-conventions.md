# 逻辑开发规范 (Logic Conventions)

在进行逻辑开发（特别是涉及列表、表单和弹窗）时，必须遵循以下规则：

## 初始化显示控制

- **规则**：列表页的列表和表单页，在初始化数据完成之前必须隐藏。
- **实现**：使用 `loaded` 状态位进行控制，确保用户不会看到“无数据”或“空表单”的中间状态。如果是分页列表，使用了`<van-list>`组件，跳过这一步。

## 表单验证与输入控制

- **全量验证**：每个表单都必须配备完整的验证逻辑。
- **即时输入控制**：对于 `input` 输入框，必须在 `input` 事件中进行内容与类型的判断和限制（例如限制只能输入数字）。

## 用卫语句取代嵌套表达式

- **规则**：在复杂的逻辑判断中，必须使用卫语句（guard clauses）来提前返回，而不是使用嵌套的 `if-else` 表达式。
  示例：

```js
function validateForm(data) {
    if (!data.name) {
        return '请输入姓名';
    }
    if (!data.email) {
        return '请输入邮箱';
    }
    // 其他验证逻辑...
    return true;
}
```
